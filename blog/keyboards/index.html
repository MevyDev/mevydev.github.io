      You don't need more than 50 keys

      When might you use a full sized keyboard

      There are quite a few situations where using a full sized keyboard is completely acceptable. Mostly when you don't expect to be doing this one thing for extended periods of time and using a numpad is the easiest.

      Here are a few examples: A game requires you to use a numpad for some reason and you don't feel like rebinding it. You are going to be working with a lot of numbers (accounting) where you don't need to type any other keys than on the numpad. Again, not for a long time. A program has some very insensible binds and you can't/don't want to rebind stuff or make a keyboard layout for this program. Alt codes... (You can try rebinding it, but good luck)

      Numpad is (usually) bad at the only thing it tries to be good at

      What would you use a numpad for? Usually some number heavy things. What if you make a mistake tho? You have to move your whole hand over to the left over to the backspace key and then right back.

      • That is like 10 keys worth of movement just for one keypress. Why is there no backspace key on the numpad? Math requires a lot of backtracking and you usually cannot afford to make a mistake in financing. (you should try to avoid typos anyways but it happens) In general, there are a lot of keys that you cannot type on a numpad that you'd want to type.
      • Parentheses, equals, delete... There exist almost no more-than-full sized keyboards (and for a good reason)

      The solution

      A separate numpad! With more keys! Doesn't constantly get in the way, is way more ergonomic... You can remove it any time!
